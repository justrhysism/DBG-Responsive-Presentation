Fluid Design

Fluid Design Also known as “Liquid” design.

Width of the design is the width of the screen no matter the width.

As the width of the screen reduces, the content “flows” to fit.

These designs are based on percentages to determine element widths, and ems to determine font sizes.

Fluid design is great, but...

  • Very wide screens it looks silly
  • Very skinny screen (e.g. mobile screens) it's unreadable

What's the solution for a fluid website which is device agnostic?

Responsive Design

Responsive Design

  • Just a concept
  • No hard and fast rules*
  • No right or wrong way
  • An accepted standard is starting to form
    • for better or for worse...
  • Don't confuse responsive design with Adaptive Design

* we will get back to rules

Adaptive Design

Adaptive design targets specific devices.

Not responsive, as they don't fit all devices.

Easy to confuse with Responsive Design, but they are not the same thing. Many templates out there call themselves responsive, when they are in fact only adaptive.

Collaboration between Designer and Developer

Current Design Process

Responsive design is hard.

Can't follow the traditional process of “design is done, code it now.”

This is an old, outdated and troublesome process.

Designs need to be created with responsiveness in mind from the beginning.

Collaborative Design

The design needs collaboration between designer and developer.

A design isn't done until the site is complete.

The site constantly cycles between design and development.

No longer can design and development be mutually exclusive.

It must be an integrated process.

Media Queries

Media Queries

Responsive design is a concept, yes. But it uses some rather old CSS and some very basic fundamentals to work: CSS Media Queries.

CSS Media Queries are just simple override instructions when the viewport meets certain requirements. 

body {
    font-size: 1em;
}

/* When the viewport is <979px... */
@media (max-width: 979px) {
    body {
        /* Reduce font size */
        font-size: 0.75em;
    }
}

Media Queries Simplified

We simply tell the browser what to do when the screen gets smaller (or bigger). 

body {
    /* Normally, do this */
}

/* Although, if we match this rule... */
@media (rule-to-match) {
   body {
        /* Do this */
   }
}

Can have as many Media Queries as is necessary. This particular presentation has 14 seperate Media Queries for different widths and heights.

Responsive Methods

Responsive Methods

There are various methods we can utilise to make a website responsive:

  • Folding
  • “Removing”
  • Hiding
  • Font sizing
  • Minor adjustments

To make a responsive layout, several of the above methods will be utilised.

Folding Method

Folding Method

Folding is a method of folding blocks of content together.

This is the most common method for creating a responsive layout.

Folding: The Code

<div class="row">
   <div class="span-half purple"></div>
   <div class="span-half red"></div>
</div>

/* Simplified */
.span-half {
    width: 49%;
    margin-left: 2%;
    float: left;
}

@media (max-width: 379px) {
    .span-half {
        float: none;
        width: 100%;
        margin-left: 0;
    }
}

Folding Method Uses

  • Text blocks
  • LHS navigation
  • Images
  • Form fields (field drops under label)
  • Banner text

Utilises natural content flow.

Very easy to implement when structured with a grid system.

Folding Method Advanced

Blocks can fold at different intervals.

“Removing” Method

“Removing” Method

This method simply “removes” the content from the screen.

“Removing” Method Issues

There are several issues with the removing method:

  • Content is still in the DOM, therefore is still being downloaded
    • If the element contains images, the device is still going to download the images, even though it's not going to display them.
  • If the content isn't worth displaying for the “mobile” site, is it worth display at all? 
<div style="display: none;">
    <!-- Contents are still downloaded -->
    <img src="/images/banner-sale.jpg" alt="50% Sale" />
    <img src="/images/banner-xmas.jpg" alt="Merry Xmas" />
</div>

“Removing” Method Uses

There are however, times when removing elements from the screen can be appropriate.

The C3 CMS site removes the "try the free demo" button from the header. This button is accessible through the menu.

While purists may suggest that it's an 8.5KB waste of bandwidth (by not displaying the image), I decided more was to be gained from removing the button than leaving it there.

Hiding Method

Hiding Method

The hiding method is not the same as the “removing” method as the elements are still accessible by the user.

This is a popular method for navigation menus, where attemping to display a large list of menu items takes up too much space, and “removing” it provides a terrible user experience.

Usually, the content is hidden and becomes accessible when the user touches a button to display the content.

Hiding Method Uses

A few examples:

  • DBG C3 CMS Site
    • Menu becomes a drop down
  • Fineline C3 Site
    • 'Facebook' app method
  • Eagles Site
    • Slide down menu - Bootstrap default

Font Sizing

Font Sizing

This one is pretty self explanatory. As the screen size decreases, the font size should decrease with it ensuring readability.

...and the other way, when a screen size increases, the font size should increase also.

Designs need to respect flow, so that font size changes don't break the design.

Font Sizing for Alternative Screens

The design should look fine at almost any font-size setting.

TV screens have a low resolution, yet set a large font size so it's readable from a distance.

The vision-impaired often increase font sizes in their browser to increase readability.

Minor Adjustments

Minor Adjustments

Usually the final step in responsive design. Done case-by-case.

As issues are discovered throughout testing, add rules to tweak the design.

The C3 CMS site does this with the RHS buttons.

The Fineline C3 site has minor adjustments to the header size before going ”mobile“.

Method Summary

Method Summary

A responsive design is made up various methods and techniques.

Never is a design based on a single method; several, if not all, methods are used to achieve a responsive design.

The keys are to respect and understand content flow, whilst utilising fluid design.

Don'ts

Don't do...

Time to trump those old habits. Old habits die hard, but they must die now.

  • Fixed anything:
    • Navigation (especially)
    • Banners
    • Triple Boxes (height)
    • Forms
  • Using px.

Don't Code Examples

When designing, think about how it's going to be coded:

/* Any of the following makes you a bad person */
form {
    width: 200px; /* Bad */
    height: 400px; /* Worse */
    font-size: 11px;
	
    /* ALWAYS overused and often unnecessary */
    float: left;
    display: block;
}

/* What it should look like */
form {
    width: 40%;
    /* No height, let it flow */
    font-size: 1.2em;
}
